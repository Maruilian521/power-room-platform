# 智慧运维中心 - 综合态势大屏使用说明

## 📊 概述

**智慧运维中心 (Integrated Operations Center)** 是针对100个配电室的统一管理驾驶舱大屏，采用**"平时看态势，战时看报警"**的设计理念，汇聚各子系统的核心数据，为运维指挥提供一站式态势感知。

### 设计风格
- **赛博科技风**：深色系主题，青色高亮，适配电力物联网行业
- **三栏包围式布局**：左侧健康态势、中央GIS地图、右侧实时指挥
- **数据实时刷新**：关键指标自动更新，告警滚动显示

---

## 🎯 功能模块详解

### 1️⃣ 顶部核心KPI指标（Top KPI Cards）

展示全局关键指标，让领导一眼放心：

| 指标 | 说明 | 数据来源 |
|------|------|----------|
| **安全运行天数** | 全网持续安全运营天数（如：128天） | 系统统计 |
| **站点在线率** | 在线站点数 / 总站点数（98/100） | 站点监控模块 |
| **今日告警总数** | 当日累计告警数（564条） | 告警与事件模块 |
| **待处理急单** | 紧急工单数量（10个，标红提示） | 工单管理模块 |
| **实时总负荷** | 全网100个站点总负荷（167.8 MW） | 实时数据汇聚 |

**交互特性**：
- 卡片悬停高亮，带青色光晕效果
- 四角装饰线条动态显示

---

### 2️⃣ 左侧：健康与环境 (Health & Environment)

#### A. 环境安防态势雷达
聚合全网环境异常站点数量：
- 🌡️ **高温预警**：显示有高温告警的站点数（如：8站）
- 💧 **水浸告警**：水浸传感器触发的站点数（如：0站）
- 🚬 **烟感告警**：烟雾检测异常站点数（如：0站）
- 🔒 **非法入侵**：门禁/视频AI识别入侵的站点数（如：2站）

**数据价值**：
- 快速识别全网环境异常热点区域
- 状态颜色映射：正常=绿色边框，预警=黄色，告警=红色

#### B. 设备健康度分布（饼图）
展示100个站点的变压器负载率分布：
- 🟢 **低负载 (<50%)**：42站 - 设备利用率低，可能存在"大马拉小车"
- 🔵 **中负载 (50%-80%)**：48站 - 理想运行区间
- 🟡 **高负载 (>80%)**：8站 - 需要关注，可能临近满载
- 🔴 **超负荷**：2站 - 需要立即扩容或负荷调度

**交互**：
- 鼠标悬停显示具体数据和百分比
- 下方图例显示每个分类的站点数量

#### C. 物资预警
来源：备品备件管理模块
- ⚠️ **库存不足**：当前有 18 类备件库存低于安全阈值
- ⏰ **即将过期**：5 类备件即将到达保质期

---

### 3️⃣ 中央：GIS站点分布态势 (Site Map Core)

#### 地图视觉设计
- **底图**：使用ECharts Geo组件绘制简化的北京区域地图
- **站点打点**：100个配电室按经纬度坐标分布
- **状态颜色映射**：
  - 🟢 **绿色** = 正常运行（带光晕阴影）
  - 🟡 **黄色** = 一般告警（闪烁光晕）
  - 🔴 **红色** = 严重告警（波纹动画）
  - ⚫ **灰色** = 离线站点（半透明）

#### 重点关注浮层（右上角）
自动展示Top 3重点站点：
- **负荷最高的2个站点**（可能即将过载）
- **温度最高的1个站点**（可能存在散热问题）

**交互**：
- 鼠标悬停站点显示详细信息：站点名、健康分、负荷、温度、湿度
- 点击站点可跳转到站点详情页（预留接口）
- 地图支持拖拽和缩放（roam: true）

#### 后续扩展
可替换为真实地图服务：
- **高德地图API**：适合国内精确定位
- **Mapbox GL JS**：自定义样式，支持深色主题
- **Cesium**：如需3D地球视角

---

### 4️⃣ 右侧：实时与指挥 (Realtime & Command)

#### A. 实时告警滚动流
**过滤规则**：只显示以下级别的告警，避免刷屏
- 🔴 **严重 (Critical)**：设备过载、明火识别、水浸等
- 🤖 **AI识别异常 (AI Anomaly)**：未戴安全帽、烟雾检测、人员倒地

**显示内容**：
- 时间戳（如：14:32:15）
- 站点名称（如：海淀区No.91配电室）
- 告警类型（标签显示，颜色区分）
- 简短描述

**交互**：
- 滚动容器，最新告警在顶部
- 严重告警背景高亮（红色半透明）
- 自动刷新（每10秒）

#### B. 工单执行漏斗
展示工单流转进度（漏斗图）：
```
新建工单 (50) ────▼
进行中 (28)   ────▼
已完成 (12)
```

**超时预警**：
- 底部显示超时未完成的工单数（10个，红色背景）

#### C. AI视觉快照
当有AI识别异常时，展示抓拍缩略图：
- 2x2网格布局（最多显示4个最新快照）
- 显示内容：
  - 图像占位符（带图标）
  - 异常类型（如：未戴安全帽）
  - 站点名称
  - 识别时间和置信度（如：98%）

**交互**：
- 点击快照可查看大图或跳转到视频监控页面

---

### 5️⃣ 底部：趋势与成效 (Trends & Effectiveness)

#### A. 全网负荷/能耗趋势（24小时曲线）
**模式切换**：
- **负荷模式**：显示每小时总负荷（MW）
- **能耗模式**：显示累计能耗（MWh）

**特点**：
- 渐变填充面积图（青绿渐变）
- 自动识别用电高峰（8-12点、18-22点）
- 鼠标悬停显示精确数值

#### B. 巡检完成率
**环形进度条**：
- 中心显示完成率（如：98.5%）
- 外圈渐变色彩（绿-青）

**详细统计**：
- 计划巡检：100站
- 已完成：98站
- 剩余：2站

**数据来源**：运维报表 - 巡检表现

## 🛠️ 技术实现

### 文件结构
```
src/
├── api/
│   └── integrated-ops.ts            # Mock数据服务
├── views/
│   └── Dashboard/
│       ├── IntegratedOpsCenter.vue  # 主组件
│       └── components/
│           ├── HealthEnvironmentPanel.vue    # 左侧面板
│           ├── SiteMapCore.vue               # 中央GIS地图
│           ├── RealtimeCommandPanel.vue      # 右侧面板
│           └── TrendsEffectivenessPanel.vue  # 底部面板
└── config/
    └── navigation.ts                # 菜单配置
```

### 数据刷新策略
| 模块 | 刷新频率 | 说明 |
|------|----------|------|
| 顶部KPI | 30秒 | 避免频繁请求 |
| 告警滚动流 | 10秒 | 保证告警及时性 |
| GIS地图 | 静态 | 站点状态随告警联动 |
| 负荷趋势 | 静态 | 每小时更新一次即可 |

### Mock数据生成
所有数据由 `integrated-ops.ts` 生成，包括：
- 100个站点的坐标、状态、负荷、温度等
- 24小时负荷曲线（模拟用电高峰）
- 实时告警流（随机生成，自动排序）
- 工单漏斗、AI快照等

### 样式主题
- 使用CSS变量 `var(--tech-primary)`, `var(--status-danger)` 等
- 自动适配 `iot-matrix` 和 `terran-command` 两种主题
- 响应式布局，支持1200px以下屏幕

## 🚀 使用方式

### 访问路径
1. 登录系统后，点击左侧菜单 **"驾驶舱"**
2. 选择 **"智慧运维中心"** 子菜单
3. 或直接访问：`/#/dashboard/integrated-ops`

### 后续优化建议
1. **地图升级**：集成高德/百度地图API，显示真实街道
2. **实时数据流**：接入WebSocket，实现告警推送
3. **数据联动**：点击地图站点 → 自动切换左侧环境数据
4. **全屏模式**：增加全屏按钮，投屏到监控中心大屏
5. **历史回溯**：支持查看过去24小时、7天的趋势数据
